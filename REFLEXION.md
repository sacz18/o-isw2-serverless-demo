1. ¿Qué tipo de error evita el CI?
El CI evita errores técnicos como fallos en tests, errores de sintaxis y problemas de compilación. También detecta conflictos de integración cuando el código de diferentes desarrolladores no funciona junto.
2. ¿Qué tipo de error NO evita?
El CI no evita errores de lógica de negocio o requisitos mal entendidos. Si el código funciona perfectamente pero hace la cosa equivocada, el CI no lo detecta. Tampoco previene problemas de usabilidad o bugs en casos que no están cubiertos por tests.
3. ¿Qué pasaría si un equipo ignora el CI?
Sin CI, el código que funciona localmente puede fallar en producción, causando bugs que llegan a los usuarios. La integración se vuelve dolorosa porque los conflictos se acumulan durante semanas. El equipo pierde confianza para hacer cambios porque nadie sabe qué se va a romper.
